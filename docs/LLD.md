## 1. Firestore Data Models

### Collection: `jobs`
```
jobs/{job_id}
├── job_id          : str          (auto UUID)
├── title           : str
├── role_description: str          (full JD text extracted from PDF)
├── jd_file_url     : str          (Firebase Storage URL)
├── headcount       : int
├── budget_min      : float
├── budget_max      : float
├── locations       : list[str]
├── experience_min  : int          (years)
├── experience_max  : int
├── team            : str          (e.g. "Backend Engineering")
├── tech_stack      : list[str]    (parsed from JD by A1)
├── status          : str          (ENUM: DRAFT | SOURCING | SCREENING | SCHEDULING | CLOSED)
├── created_at      : timestamp
└── updated_at      : timestamp
```

### Collection: `jobs/{job_id}/candidates`
```
candidates/{candidate_id}
├── candidate_id        : str
├── job_id              : str
├── name                : str
├── email               : str
├── github_url          : str
├── linkedin_url        : str | null
├── location            : str
├── pipeline_stage      : str   (ENUM below)
│   SOURCED → OA_SENT → BEHAVIORAL_COMPLETE
│   → SCORED → SHORTLISTED → INTERVIEW_SCHEDULED
│   → INTERVIEW_DONE → OFFERED → HIRED | REJECTED
│
├── source_score        : float         (0–100, from A1)
├── github_signals      : map
│   ├── languages       : list[str]
│   ├── top_repos       : list[str]
│   ├── commit_frequency: str
│   └── profile_summary : str
│
├── oa_link             : str           (unique URL for candidate OA)
├── oa_token            : str           (UUID for auth-free access)
├── oa_questions        : list[map]     (generated by A2)
│   └── { question_id, question_text, type: MCQ|CODING|TEXT, options? }
├── oa_responses        : list[map]     (submitted by candidate)
│   └── { question_id, answer, submitted_at }
├── oa_submitted_at     : timestamp | null
│
├── behavioral_questions: list[str]     (generated by A2)
├── behavioral_transcript: list[map]
│   └── { role: "assistant"|"candidate", content: str, timestamp }
├── behavioral_complete : bool
│
├── composite_score     : float         (0–100, from A5)
├── score_breakdown     : map
│   ├── source_fit      : float
│   ├── oa_score        : float
│   └── behavioral_score: float
├── rank                : int           (within this job's candidate pool)
├── shortlisted         : bool
│
├── calendly_link       : str | null
├── zoom_url            : str | null
├── interview_slot      : timestamp | null
├── interview_status    : str   (PENDING | CONFIRMED | RESCHEDULED | COMPLETED)
│
├── referral_id         : str | null    (if sourced via referral)
├── created_at          : timestamp
└── updated_at          : timestamp
```

### Collection: `jobs/{job_id}/salary_report`
```
salary_report/
├── job_id          : str
├── location        : str
├── role_title      : str
├── p25             : float
```

```
email_logs/{log_id}
```

### Collection: `referrals`

```
referrals/{referral_id}
```

## 4. Agent Function Signatures

### A1 — Sourcing Hunter (`a1_sourcing_hunter.py`)
```python
async def run_sourcing_hunter(state: SourcingState) -> SourcingState:
    """
    Entry node for Graph 1.
    1. parse_jd(state.jd_text) -> tech_stack, experience_range, locations
    2. search_github(tech_stack, experience_range) -> raw_profiles[]
    3. score_candidates(raw_profiles, jd_embedding) -> scored_candidates[]
    4. persist_to_firestore(job_id, scored_candidates)
    5. return state with sourced_candidates populated
    """

def parse_jd(jd_text: str) -> dict:
    """LLM call: extract tech_stack[], experience_range, locations[] from raw JD."""

def search_github(
    tech_stack: list[str],
    location: str,
    experience_keywords: list[str],
    max_results: int = 50
) -> list[GithubProfile]:
    """
    GitHub Search API: GET /search/users?q={tech_stack}&type=Users&per_page=50
    Then GET /users/{login} and GET /users/{login}/repos
    Returns structured GithubProfile list.
    """

def score_source_fit(
    candidate_profile: GithubProfile,
    jd_embedding: list[float]
) -> float:
    """
    Embed candidate's bio + top repo READMEs.
    Cosine similarity vs JD embedding. Returns 0-100.
    """

class GithubProfile(TypedDict):
    login: str
    name: str
    email: str | None
    location: str
    bio: str
    top_repos: list[str]
    languages: list[str]
    public_repos: int
    followers: int
    github_url: str
```

### A2 — Behavioral Chat + OA Generator (`a2_behavioral_oa.py`)
```python
async def run_behavioral_oa(state: ScreeningState) -> ScreeningState:
    """
    1. generate_oa_questions(jd_text, tech_stack) -> OAQuestion[]
    2. generate_behavioral_questions(jd_text) -> str[]
    3. persist oa_questions + behavioral_questions to Firestore
    4. return state with oa_questions populated (A6 will send the OA link)
    """

def generate_oa_questions(
    jd_text: str,
    tech_stack: list[str],
    n_mcq: int = 5,
    n_coding: int = 2,
    n_text: int = 3
) -> list[OAQuestion]:
    """
    LLM prompt: 'Given this JD, generate {n_mcq} MCQ, {n_coding} coding,
    {n_text} open-ended questions testing: {tech_stack}'
    Returns structured OAQuestion list.
    """

def generate_behavioral_questions(
    jd_text: str,
    n: int = 5
) -> list[str]:
    """
    LLM prompt: 'Generate {n} STAR-format behavioral questions for this role.'
    """

async def conduct_behavioral_chat(
    candidate_id: str,
    questions: list[str],
    candidate_message: str,
    history: list[dict]
) -> str:
    """
    LLM call with history. Conducts conversational behavioral assessment.
    Asks follow-up questions. Returns next agent message.
    """

def evaluate_oa_responses(
    questions: list[OAQuestion],
    responses: list[OAResponse]
) -> float:
    """
    LLM evaluates each answer (correctness, depth, clarity).
    Returns normalized OA score 0-100.
    """
```

### A3 — Interview Coordinator (`a3_interview_coordinator.py`)
```python
async def run_coordinator(state: ScreeningState) -> ScreeningState:
    """
    1. create_calendly_link(candidate_id, job_id) -> calendly_link
    2. create_zoom_meeting(candidate_name, slot) -> zoom_url
    3. update state with links
    4. trigger A6 to send invite email
    5. poll invite_status from A6 (accepted / rescheduled / no-reply)
    """

def create_calendly_scheduling_link(
    event_type_uuid: str,
    prefill_name: str,
    prefill_email: str
) -> str:
    """
    POST Calendly API: /scheduling_links
    Returns unique one-time booking URL.
    """

def create_zoom_meeting(
    topic: str,
    start_time: datetime,
    duration_minutes: int = 60
) -> dict:
    """
    POST Zoom API: /users/me/meetings
    Returns { join_url, meeting_id, password }
    """

def handle_reschedule(
    candidate_id: str,
    new_slot: datetime,
    zoom_meeting_id: str

    PATCH Zoom: /meetings/{meetingId}
    Update Firestore interview_slot.
    """
```

### A4 — Market Analyst (`a4_market_analyst.py`)

```python
async def run_market_analyst(state: CoordinationState) -> CoordinationState:

    1. fetch_salary_benchmarks(role_title, locations, tech_stack)
    2. compute_percentiles(raw_data)
    3. check_budget_warning(budget_max, p25)

def fetch_salary_benchmarks(
    role_title: str,

    tech_stack: list[str],
    experience_years: tuple[int, int]

    """
    Aggregates from:
    - Levels.fyi (API or scraped dataset)
    - BLS Occupational Employment Stats (public API)
    - LinkedIn Salary Insights (mocked)
    Returns raw salary data points.
    """

def compute_percentiles(salary_data: list[float]) -> SalaryReport:
    """
    numpy.percentile([25, 50, 75, 90])
    Returns SalaryReport TypedDict.
    """

def check_budget_warning(budget_max: float, p25: float) -> bool:
    """Returns True if budget_max < p25 (below market floor)."""
```

### A5 — Scoring & Ranking Engine (`a5_scoring_ranking.py`)
```python
async def run_scoring_engine(state: ScreeningState) -> ScreeningState:
    """
    1. compute_composite_score(source_score, oa_score, behavioral_score)
    2. update candidate record in Firestore
    3. recompute_rankings(job_id) — rank all candidates for this job
    4. set shortlisted = True if rank <= headcount * 3
    5. return updated state
    """

def compute_composite_score(
    source_score: float,      # weight: 20%
    oa_score: float,          # weight: 40%
    behavioral_score: float   # weight: 40%
) -> tuple[float, ScoreBreakdown]:
    """
    Weighted average. All inputs normalized to 0-100.
    Returns (composite_score, breakdown).
    """

def evaluate_behavioral_transcript(
    transcript: list[dict],
    jd_text: str
) -> float:
    """
    LLM evaluates full transcript against rubric:
    - Communication clarity
    - Situational judgement
    - Role alignment
    Returns 0-100.
    """

async def recompute_rankings(job_id: str) -> None:
    """
    Fetches all candidates for job_id from Firestore.
    Sorts by composite_score descending.
    Writes rank (1, 2, 3...) back to each candidate document.
    """
```

## 5. LangGraph Node & Edge Definitions

### Graph 1 — Sourcing
```python
from langgraph.graph import StateGraph, END

graph1 = StateGraph(SourcingState)

graph1.add_node("sourcing_hunter", a1_sourcing_hunter.run)
graph1.add_node("behavioral_oa_init", a2_behavioral_oa.init_for_candidates)

graph1.add_edge("sourcing_hunter", "behavioral_oa_init")
graph1.add_edge("sourcing_hunter", "initial_scoring")
graph1.add_edge("sourcing_hunter", "send_outreach")

# All three nodes run in parallel after sourcing_hunter
graph1.add_edge("behavioral_oa_init", END)
graph1.add_edge("initial_scoring", END)
graph1.add_edge("send_outreach", END)
```

### Graph 2 — Screening
```python
graph2 = StateGraph(ScreeningState)
```

## 7. FastAPI Route Specs

### `POST /api/jobs`
```
Request body:  JobCreateRequest (multipart/form-data: fields + jd_file PDF)
Response:      { job_id: str, status: "SOURCING" }
Side effects:
  - Upload JD PDF to Firebase Storage
  - Parse JD text with A1
  - Persist job doc to Firestore
  - Invoke graph1.ainvoke(SourcingState) as BackgroundTask
```

### `GET /api/jobs/{job_id}/candidates`
```
Response: {
  job_id: str,
  candidates: CandidateResponse[],   # sorted by rank ascending
  total: int,
  shortlisted_count: int
}
```

### `GET /api/jobs/{job_id}/report`
```
Response: {
  salary_report: SalaryReport,
  top_candidates: CandidateResponse[5],
  audit_summary: { total_sourced, screened, shortlisted, scheduled }
}
```

### `POST /api/oa/{oa_token}/submit`
```
# Public endpoint — no auth. oa_token validates identity.
Request:  OASubmission
Response: { received: true, submitted_at: datetime }
Side effects:
  - Persist oa_responses to Firestore
  - Trigger A5 to re-score, then A6 to send result email
```

### `GET /api/chat/{candidate_id}/start`
```
# Returns the FAQ chat context for the candidate
Response: {
  candidate_name: str,
  role_title: str,
  pipeline_stage: str,
  history: list[ChatMessage]
}
```

### `POST /api/chat/{candidate_id}/message`
```
Request:  ChatMessage { content: str }
Response: ChatReply   { content: str, timestamp: datetime }
Side effects: Appends to faq_threads/{candidate_id} in Firestore
```

### `WS /ws/pipeline/{job_id}`
```
# Server pushes events on candidate stage changes
Event shape: {
  type: "STAGE_CHANGE" | "AGENT_UPDATE" | "SCORE_UPDATE",
  candidate_id: str,
  new_stage: str,
  agent: str,
  timestamp: str
}
```
